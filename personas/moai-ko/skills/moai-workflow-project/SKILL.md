# MoAI 워크플로우 프로젝트 - 통합 프로젝트 관리 시스템

목적: 지능적인 자동화와 Claude Code 통합을 갖춘 통합 아키텍처로 문서 생성, 다국어 지원, 템플릿 최적화를 통합하는 포괄적인 프로젝트 관리 시스템.

범위: 초기화부터 유지 관리까지 완전한 프로젝트 라이프사이클을 지원하는 단일 응집 시스템으로 문서 관리, 언어 초기화, 템플릿 최적화를 통합합니다.

대상: 프로젝트 설정, 문서 생성, 다국어 지원, 성능 최적화를 위한 Claude Code 에이전트.

---

## 빠른 참조

핵심 기능:

- 문서 관리: 다국어 지원을 갖춘 템플릿 기반 문서 생성
- 언어 초기화: 언어 감지, 설정, 지역화 관리
- 템플릿 최적화: 고급 템플릿 분석 및 성능 최적화
- 통합 인터페이스: 모든 기능을 통합하는 단일 진입점

주요 기능:

- 자동 프로젝트 유형 감지 및 템플릿 선택
- 영어, 한국어, 일본어, 중국어를 지원하는 다국어 문서 생성
- 성능 벤치마킹을 갖춘 지능적인 템플릿 최적화
- SPEC 기반 문서 업데이트
- Markdown, HTML, PDF를 포함한 다중 형식 내보내기

지원되는 프로젝트 유형:

- 웹 애플리케이션
- 모바일 애플리케이션
- 커맨드라인 인터페이스 도구
- 라이브러리 및 패키지
- 머신러닝 프로젝트

## 구현 가이드

### 모듈 아키텍처

문서 관리 기능:

- 템플릿 기반 문서 생성
- 웹, 모바일, CLI, 라이브러리, ML 프로젝트를 위한 프로젝트 유형 감지
- 지역화된 콘텐츠를 갖춘 다국어 지원
- 자동 업데이트를 위한 SPEC 데이터 통합
- 다중 형식 내보내기 기능

언어 초기화 기능:

- 프로젝트 콘텐츠에서 자동 언어 감지
- 포괄적인 언어 설정 관리
- 비용 최적화를 갖춘 에이전트 프롬프트 지역화
- 도메인별 언어 지원
- 로케일 관리 및 문화적 적응

템플릿 최적화 기능:

- 복잡성 지표를 갖춘 고급 템플릿 분석
- 크기 축소를 통한 성능 최적화
- 지능적인 백업 및 복구 시스템
- 벤치마킹 및 성능 추적
- 자동화된 최적화 권장사항

### 핵심 워크플로우

완전한 프로젝트 초기화 워크플로우:

1단계: 프로젝트 디렉토리 경로를 지정하여 프로젝트 관리 시스템을 초기화합니다.

2단계: 다음 설정 파라미터로 완전한 설정을 실행합니다:

- 언어 설정: 영어는 "en", 한국어는 "ko"와 같은 기본 언어 코드 지정
- 사용자 이름: 개인화를 위한 개발자 또는 팀 이름 제공
- 도메인: backend, frontend, mobile과 같은 프로젝트 도메인 목록
- 프로젝트 유형: web_application과 같은 프로젝트 유형 지정
- 최적화 활성화: 초기화 중 템플릿 최적화를 활성화하려면 true로 설정

3단계: 다음을 포함하는 초기화 결과를 검토합니다:

- 토큰 비용 분석을 갖춘 언어 설정
- 문서 구조 생성 상태
- 템플릿 분석 및 최적화 보고서
- 다국어 문서 설정 확인

SPEC 워크플로우에서 문서 생성:

1단계: 다음 구조로 SPEC 데이터를 준비합니다:

- 식별자: SPEC-001과 같은 고유한 SPEC ID
- 제목: 기능 또는 컴포넌트 제목
- 설명: 구현에 대한 간략한 설명
- 요구사항: 특정 요구사항 목록
- 상태: Planned, In Progress, Complete와 같은 현재 상태
- 우선순위: High, Medium, Low와 같은 우선순위 레벨
- API 엔드포인트: path, method, description을 포함하는 엔드포인트 정의 목록

2단계: SPEC 데이터에서 포괄적인 문서를 생성합니다.

3단계: 다음을 포함하는 생성된 문서를 검토합니다:

- 요구사항을 갖춘 기능 문서
- 엔드포인트 세부 정보를 갖춘 API 문서
- 업데이트된 프로젝트 문서 파일
- 설정된 경우 다국어 버전

템플릿 성능 최적화 워크플로우:

1단계: 지표를 수집하기 위해 현재 템플릿을 분석합니다.

2단계: 최적화 옵션을 설정합니다:

- 먼저 백업: 최적화 전 백업 생성에 true로 설정
- 크기 최적화 적용: 파일 크기를 줄이기 위해 활성화
- 성능 최적화 적용: 로딩 시간을 개선하기 위해 활성화
- 복잡성 최적화 적용: 템플릿 구조를 단순화하기 위해 활성화
- 기능성 보존: 모든 기능이 그대로 유지되도록 보장

3단계: 최적화를 실행하고 결과를 검토합니다:

- 달성된 크기 감소 비율
- 성능 개선 지표
- 백업 생성 확인
- 상세 최적화 보고서

### 언어 및 지역화

자동 언어 감지 프로세스:

시스템은 다음 방법을 사용하여 언어 지표에 대한 프로젝트를 분석합니다:

- 주석 및 문자열을 검토하는 파일 콘텐츠 분석
- 로케일 설정에 대한 설정 파일 검토
- 시스템 로케일 감지
- 디렉토리 구조 패턴

다국어 문서 구조:

여러 언어로 문서를 생성할 때 시스템은 다음을 생성합니다:

- 한국어는 docs/ko, 영어는 docs/en과 같은 언어별 문서 디렉토리
- 언어 협상 설정
- 언어 버전 간 자동 리디렉션 설정

에이전트 프롬프트 지역화:

지역화 시스템은 다음을 제공합니다:

- 에이전트를 위한 언어별 지침
- 문화적 맥락 적응
- 다국어 프롬프트를 위한 토큰 비용 최적화 권장사항

### 템플릿 최적화

성능 분석 프로세스:

템플릿 분석은 다음 지표를 제공합니다:

- 파일 크기 및 복잡성 측정
- 성능 병목 식별
- 최적화 기회 점수
- 리소스 사용 패턴
- 백업 권장사항

지능적인 최적화 프로세스:

최적화 시스템은 다음 기법을 적용합니다:

- 공백 및 중복성 감소
- 템플릿 구조 최적화
- 복잡성 감소 기법
- 성능 캐싱 개선

### 설정 관리

통합 설정 상태:

프로젝트 상태에는 다음이 포함됩니다:

- 프로젝트 메타데이터 및 유형 분류
- 언어 설정 및 관련 비용
- 문서 완성 상태
- 템플릿 최적화 결과
- 모듈 초기화 상태

언어 설정 업데이트 프로세스:

언어 설정을 업데이트할 때 다음 파라미터를 설정합니다:

- 대화 언어: 사용자 대면 응답을 위한 언어
- 에이전트 프롬프트 언어: 내부 에이전트 지침을 위한 언어, 비용 최적화를 위해 영어로 유지되는 경우가 많음
- 문서 언어: 생성된 문서를 위한 언어

업데이트는 다음 자동 변경을 트리거합니다:

- 설정 파일 수정
- 문서 구조 업데이트
- 템플릿 지역화 조정

## 리소스

### 설정 구조

프로젝트 설정 필드:

- 프로젝트 이름: 프로젝트의 표시 이름
- 프로젝트 유형: web_application, mobile_application, cli_tool과 같은 분류
- 초기화 타임스탬프: 프로젝트가 초기화된 시간
- 언어 설정: 대화, 에이전트 프롬프트, 문서 언어 설정
- 시스템 버전: 프로젝트 관리 시스템 버전
- 초기화 상태: 모든 모듈이 완전히 초기화되었는지 여부

지원되는 각 언어에 대한 언어 설정 필드:

- 이름: 영어로 된 표시 이름
- 기본 이름: 기본 언어로 된 표시 이름
- 코드: en, ko, ja, zh와 같은 언어 코드
- 로케일: 시스템 로케일 문자열
- 에이전트 프롬프트 언어: 영어 또는 지역화된 프롬프트 사용 여부
- 토큰 비용 영향: 영어 외 프롬프트의 토큰 사용량 증가율

### 성능 지표

문서 생성 성능:

- 완전한 문서 생성은 일반적으로 2~5초 내에 완료
- 언어 감지 분석은 약 500밀리초 내에 완료
- 템플릿 최적화 기간은 프로젝트 크기에 따라 10~30초 사이에 다름
- 설정 업데이트는 약 100밀리초 내에 완료

메모리 사용 특성:

- 기본 시스템은 약 50MB RAM 필요
- 대규모 프로젝트는 템플릿 수에 따라 추가로 10~50MB가 필요할 수 있음
- 최적화 캐시는 성능 향상을 위해 약 5~20MB 사용

파일 크기 영향:

- 문서 파일은 프로젝트당 50~200KB 범위
- 최적화 백업은 원본 템플릿의 크기와 일치
- 설정 파일은 완전한 프로젝트 설정에 5~10KB 범위
